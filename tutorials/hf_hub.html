Hugging Face Hub

허깅페이스 헤커톤은 컬랙션(collection)과 스페이스(space)로 구성됩니다. 컬랙션은 스페이스, 모델 등 다양한 허깅페이스 리소스가 모인 집합입니다. 참가자의 허깅페이스 스페이스는 아래과 같이 Hugging Face KREW Hackathon 2023이라는 이름의 컬랙션에 포함됩니다.

"Hugging Face KREW Hackathon 2023" (collection)
    ├── 참가자프로젝트(1) (space)
    ├── ...
    └── 참가자프로젝트(n) (space)

1. 허깅페이스 스페이스 생성

참가자는 Pseudo Lab 컬랙션 우측 상단의 New 버튼을 누르고 Space를 선택해서 자신의 프로젝트 스페이스를 생성합니다.

스페이스 생성 페이지에서는 프로젝트 스페이스 생성을 위해서 필요한 정보를 작성합니다. 하나씩 확인해보겠습니다.

Space Name 과 License

스페이스 생성 초기 화면에서 Space name과 License를 입력합니다.

Space name은 헤커톤 컬랙션에 카드 이름으로 표시됩니다. Licsnse는 아파치 라이선스를 사용하기 위해서 “Apache License”를 선택합니다.

SDK 와 Space hardware

이제 SDK와 Space hardware를 항목에서 프로젝트 스페이스의 실행을 위한 설정을 진행합니다.

SDK 는 스페이스 데모 구성을 위해 어떤 기능을 사용할 지 결정합니다. 자신이 스페이스를 구성할 때 사용할 SDK를 선택합니다. 모든 SDK가 생소하신가요? 그렇다면 쉽게 배울 수 있고 다양한 예제가 있으니 허깅페이스에서 제공하는 UI 인터페이스 Gradio를 선택하길 추천드립니다. 튜토리얼에 Gradio에 대한 가이드 링크를 확인할 수 있습니다.

Space hardware는 프로젝트 스페이스를 실행할 하드웨어 자원을 기본 Free 항목으로 선택합니다. 헤커톤에서는 Credit 이 지원하는 범위에서 하드웨어 자원을 지원합니다. 프로젝트 개발 중 GPU 자원이 필요하다면 스페이스에서 변경이 가능합니다. 헤커톤에서는 Credit이 지원되는 범위에서 고급 하드웨어 자원을 지원합니다.

- 고급 GPU 하드웨어 자원을 사용하면 헤커톤 전체 Credit이 소모됩니다.
- 모든 헤커톤 참가자가 하드웨어 자원을 사용할 수 있도록 효율적으로 사용부탁드립니다.

2. 헤커톤 컬렉션과 연결

멋집니다! 이제 여러분은 자신의 프로젝트를 실행할 스페이스를 생성했고 다른 참가자들에게 빨리 소개하고 싶습니다. 프로젝트를 헤커톤 컬랙션에 연결하는 방법을 알아 봅시다.

자신의 스페이스 우측 상단의 ... 버튼을 누르고 Add to collection을 선택합니다.

Hugging Face KREW Hackathon 2023 (pseudolab)을 선택하여 자신의 스페이스를 헤커톤 컬렉션에 추가합니다.

3. 스페이스 복사해서 추가하기

자신의 Profile 상에서 생성한 스페이스를 헤커톤 컬랙션에 연결할 수 있습니다.

스페이스 우측 상단의 … 버튼 > Duplicate this Space를 선택합니다. 스페이스의 Owner를 pseudolab으로 변경하면 스페이스가 복제됩니다. 복제가 완료되면 스페이스의 조작이 pseudolab으로 변경됩니다.

이제 스페이스가 pseudolab 에 생성되었기 때문에 컬렉션에 추가 할 수 있습니다. 2. 헤커톤 컬렉션과 연결 을 참고해서 컬랙션에 추가하세요.
